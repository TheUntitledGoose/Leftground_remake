Disable managed extentions with crosh and VMC

This method was found by vk6#7391. This has only been tested to work in the latest version of 111, but it probably works on pretty much any ChromeOS version.

Prerequsites:

  • A managed Chromebook with Crosh enabled
  • The VmManagementCliAllowed policy either unset or set to true
  • A willingness to powerwash

Instructions:

  • Back up any data you want before the powerwash.
  • Powerwash by using ESC + Refresh + Power (Not sure if you need dev mode)
  • Log into your Google account as normal, but immediately disable your internet right after you sign in.
  • You should be logged into your account, but without any extensions installed due to being offline.
  • Open up crosh by hitting CTRL+ALT+T.
  • For each extension you want to disable, run this command: vmc create-extra-disk --size 1 /home/chronos/user/Extensions/"place extension id here"
  • Re-enable your internet, and if you typed/pasted in the extension IDs correctly, those specific extensions should be blocked from ever being installed.

The way this exploit works is with this Captive Portal Authorization (CAPP) window. This is not very common to come across, but it's used for when Wi-Fi networks want you to log in. This captive portal window runs without extensions, in a partial incognito/guest instance. The silly part is that if you open a link in a new tab, it will create a new Chrome instance without extensions. This is like the v81 incog and kiosk exploits. The general formula for the background page is chrome-extension: //' + your extension id + '/ _generated_background_page.html. You can get the extension id by going to chrome://extensions and clicking details by your extension and you'll see your id right after id= in the url. The background page should be white

  • (- Make sure your bookmarks bar is open CTRL+Shift+B) - Create a new bookmark with the url chrome://kill
  • While you are on the background page click on the bookmark
